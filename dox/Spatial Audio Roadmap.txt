# 📄 `masterplan.md (Spatial Audio – Perceptual Engine Integration)`

---

## 1. Feature Overview

The Spatial Audio system is a **perceptual audio enhancement layer** that simulates a wider, more immersive soundstage using **EQ shaping and supported Android audio effects**, without requiring access to raw audio streams.

This feature operates **on top of the existing AutoEQ + user tuning system** and enhances the listening experience while remaining fully compatible with apps like Spotify and YouTube Music.

---

## 2. Core Objective

Deliver a **“3D-like immersive audio feel”** using only system-allowed processing, ensuring:

* Compatibility across Android devices
* No need for root or custom players
* Immediate perceptual impact for users

---

## 3. Product Philosophy

> 🎧 “Users don’t care about DSP — they care about how wide and immersive music feels.”

This system:

* Simulates spatial depth using psychoacoustic tricks
* Prioritizes perception over technical purity
* Keeps controls simple and intuitive

---

## 4. System Architecture

### 🔗 Audio Processing Pipeline

```text
Audio Session
   ↓
AutoEQ / User EQ (10-band system)
   ↓
Spatial Engine (Perceptual Layer)
   ↓
Final Output
```

---

### 🧠 Processing Formula

```text
finalEQ = AutoEQ or UserPreset
spatialEQ = applySpatialProfile(finalEQ, spatialLevel)
```

---

## 5. Spatial Engine Design

### 🎯 Approach

Instead of real DSP:

> Use **frequency shaping + subtle audio effects** to simulate spatial perception

---

### 🧩 Core Techniques

#### 1. Frequency Separation (Width Illusion)

* Slight treble boost
* Slight mid reduction
* Creates “air” and openness

---

#### 2. Depth Simulation

* Reduce mid frequencies slightly
* Enhance upper bands
* Pushes sound “forward” and “outward”

---

#### 3. Bass Tightening

* Slight bass reduction
* Prevents muddy, closed sound

---

#### 4. Optional Reverb Layer

* Use Android `PresetReverb` if available
* Apply only at higher intensity levels
* Keep subtle to avoid distortion

---

## 6. Spatial Profiles System

### 🎚️ Discrete Levels (Recommended)

| Level | Name       | Description           |
| ----- | ---------- | --------------------- |
| 0     | Off        | No spatial processing |
| 1     | Subtle     | Light widening        |
| 2     | Light      | Noticeable separation |
| 3     | Balanced   | Default immersive     |
| 4     | Strong     | Wide soundstage       |
| 5     | Aggressive | Fun / exaggerated     |

---

### 🧠 Profile Structure

Each level defines:

```text
bass_adjustment
low_mid_adjustment
high_mid_adjustment
treble_adjustment
reverb_level (optional)
```

---

### 📊 Example (Conceptual)

#### Level 3 (Balanced)

* Bass: -1 dB
* Low-mid: -2 dB
* High-mid: +1.5 dB
* Treble: +2 dB

👉 Result: wider, clearer sound without harshness

---

## 7. Integration with Existing EQ System

### 🔗 Processing Order (CRITICAL)

```text
2. Apply intensity scaling
3. Apply spatial profile
4. Clamp values
5. Send to audio engine
```

### ⚠️ Important Rule

> Spatial processing ALWAYS happens AFTER intensity scaling
